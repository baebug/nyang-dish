🌠주의할점

Jenkins Pipeline Script에서 아래와 같은 코드가 있다면 다음과 같은 순서로 실행해야 한다.

  1. Jenkins Pipeline Script 에서 최초 실행 시 아래 주석된 두 줄을 해제하고 실행한다.
  1. /var/lib/jenkins/workspace 에서 해당 Item의 폴더에 들어가서 관련 env파일 및 있어야 하는 파일들을 넣는다.
  1. 다시 Jenkins Pipeline Script에서 주석 해제했던 두 줄을 다시 주석처리하고 build한다.
stage('Checkout') {
    steps {
        withCredentials([usernamePassword(credentialsId: 'nyang', usernameVariable: 'GIT_USERNAME', passwordVariable: 'GIT_PASSWORD')]) {
          // 최초 실행 시, 아래 두줄의 주석을 해제하고 총 다섯 줄을 실행 뒤에
				  // 그 이후로는 아래 두줄을 주석처리하고 사용
				  //sh 'git init' // 현재 디렉토리를 Git 저장소로 초기화합니다.
          //sh 'git remote add origin https://${GIT_USERNAME}:${GIT_PASSWORD}@lab.ssafy.com/s08-final/S08P31E203.git' // 원격 저장소를 추가합니다.
          sh 'git fetch --depth=1 origin develop' // 브랜치의 최신 변경 사항을 가져옵니다.
          sh 'git sparse-checkout set ai-server-2' // 필요한 디렉토리만 가져옵니다.
          sh 'git checkout origin/develop' // 가져온 디렉토리를 checkout합니다.
        }
    }
}

🌠Dev

✨front-dev

  • Pipeline script
    pipeline {
    agent any

    stages {
        stage('Checkout') {
            steps {
                // Get some code from a GitHub repository
                withCredentials([usernamePassword(credentialsId: 'ssafy', usernameVariable: 'GIT_USERNAME', passwordVariable: 'GIT_PASSWORD')]) {
                  // 최초 실행 시, 아래 두줄의 주석을 해제하고 총 다섯 줄을 실행 뒤에
				  // 그 이후로는 아래 두줄을 주석처리하고 사용
				  //sh 'git init' // 현재 디렉토리를 Git 저장소로 초기화합니다.
                  //sh 'git remote add origin https://${GIT_USERNAME}:${GIT_PASSWORD}@lab.ssafy.com/s08-final/S08P31E203.git' // 원격 저장소를 추가합니다.
                  sh 'git fetch --depth=1 origin develop' // 브랜치의 최신 변경 사항을 가져옵니다.
                  sh 'git sparse-checkout set frontend' // 필요한 디렉토리만 가져옵니다.
                  sh 'git checkout origin/develop' // 가져온 디렉토리를 checkout합니다.
                }
            }
        }
        
        stage('Remove') {
            steps {
                script {
                    try {
                        sh "docker stop front-dev && docker network disconnect dev front-dev && docker rm front-dev && docker rmi front-dev-image"
                    }
                    catch(err) {
                        echo err.getMessage()
                        echo "front-dev container not exists"
                    }
                }
            }
        }
        
        stage('Build Image') {
            steps {
                dir("frontend"){
                    sh "docker build -t front-dev-image ."
                }
            }
        }
        
        stage("run"){
            steps{
                dir("frontend"){
                    sh "docker run -e TZ=Asia/Seoul --env-file front-dev.env -d -p 3000:3000 --network dev --name front-dev front-dev-image"
                }
            }
        }
    }
}

  stages {
   stage('Remove') {

        script {
          try {

      }
    }
  }

}
